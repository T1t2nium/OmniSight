# AI Visual Conversation Assistant — 功能需求规格

## 项目概述

一款基于浏览器的 AI 视觉对话助手。用户通过摄像头和麦克风与 AI 进行实时语音+视觉对话，AI 能理解摄像头画面内容并结合用户语音给予自然语言回复。

---

## 功能需求

### F1. 摄像头视频采集
- **F1.1** 浏览器调用 `getUserMedia` API 获取摄像头视频流
- **F1.2** 支持选择前后摄像头（如设备有多个摄像头）
- **F1.3** 本地实时预览摄像头画面
- **F1.4** 支持开启/关闭摄像头（运行时切换）
- **F1.5** 关闭摄像头后，对话降级为纯语音模式

### F2. 麦克风音频采集
- **F2.1** 浏览器调用 `getUserMedia` API 获取麦克风音频流
- **F2.2** 支持开启/关闭麦克风（运行时切换）
- **F2.3** 音频以 PCM16 格式采集，采样率 16kHz

### F3. 语音活动检测（VAD）
- **F3.1** 浏览器端实时检测用户是否在说话
- **F3.2** 检测到语音开始时，标记 speech_start 事件
- **F3.3** 检测到语音结束时，标记 speech_end 事件并附带累积的音频数据
- **F3.4** 过滤过短的噪音（< 0.3 秒）
- **F3.5** VAD 状态实时可视化反馈

### F4. 实时通信管道
- **F4.1** 前端与后端通过 WebSocket 建立持久连接
- **F4.2** 音频数据（PCM16）通过 WebSocket 流式发送
- **F4.3** 视频帧（JPEG）通过 WebSocket 发送（3-5 fps）
- **F4.4** WebSocket 断开后自动重连（指数退避）
- **F4.5** 连接状态实时展示

### F5. AI 视觉理解
- **F5.1** AI 模型接收视频帧 + 用户语音转写文本
- **F5.2** AI 能描述摄像头画面中的物体、人物、场景
- **F5.3** AI 能结合视觉上下文回答用户问题
- **F5.4** 支持多轮对话，保持上下文记忆

### F6. 语音识别（STT）
- **F6.1** 将用户语音（PCM16）转写为文本
- **F6.2** 支持中英文识别
- **F6.3** 转写结果发送给 AI 模型作为输入
- **F6.4** 转写结果在前端对话记录中展示

### F7. 语音合成（TTS）
- **F7.1** 将 AI 文本回复合成为语音
- **F7.2** 语音通过浏览器扬声器播放
- **F7.3** 支持流式输出（AI 边说边合成）
- **F7.4** 支持用户打断（用户开始说话时停止播放）

### F8. 对话管理
- **F8.1** 开始/停止对话
- **F8.2** 对话记录展示（用户语音转写 + AI 回复）
- **F8.3** 对话记录自动滚动至最新

### F9. 用户打断（Barge-in）
- **F9.1** AI 说话时用户可打断
- **F9.2** 打断后 AI 立即停止输出
- **F9.3** 打断后开始处理新的用户输入

### F10. Agent 系统
- **F10.1** 支持可扩展的 Agent 框架（BaseAgent ABC + AgentRegistry）
- **F10.2** 默认「视觉聊天伴侣」Agent，保持现有对话体验
- **F10.3** Agent 可注入不同的 system prompt，改变 AI 行为和人设
- **F10.4** 前端 Agent 选择器（玻璃态胶囊标签），支持运行时查看当前 Agent
- **F10.5** 每会话独立选择 Agent，互不干扰

### F11. UI/UX 设计
- **F11.1** 采用暗色主题（GitHub Dark），视觉统一一致
- **F11.2** 响应式布局：支持桌面（960px）、平板（768px）、手机（480px）
- **F11.3** 聊天气泡：用户/AI 区分明显，流式文本动画，时间戳 hover 显示
- **F11.4** 状态可视化：VAD 呼吸灯动画、连接状态颜色梯度、延迟实时显示
- **F11.5** 磨砂玻璃效果：状态栏和控制栏半透明 backdrop-filter
- **F11.6** 按钮交互：hover 高亮、active 缩放反馈、focus-visible 轮廓
- **F11.7** 动画尊重 prefers-reduced-motion 用户偏好
- **F11.8** 无障碍：ARIA 标签、role 属性、键盘导航支持

## 非功能需求

### N1. 性能
- **N1.1** 端到端延迟（用户说完 → AI 开始回复）< 5 秒（本地模型）
- **N1.2** 视频预览帧率 ≥ 15 fps
- **N1.3** AI 输入帧率 1-4 fps

### N2. 兼容性
- **N2.1** 支持 Chrome 90+、Edge 90+、Firefox 90+
- **N2.2** 支持 Windows 10/11

### N3. 可靠性
- **N3.1** WebSocket 断开自动重连
- **N3.2** AI 模型不可用时优雅降级
- **N3.3** 30 分钟连续对话无内存泄漏

### N4. 安全性
- **N4.1** 摄像头/麦克风权限仅在用户授权后使用
- **N4.2** 本地模型模式下所有数据不离开本机
- **N4.3** 无硬编码密钥，敏感配置通过 .env 管理

### N5. 可维护性
- **N5.1** AI 模型提供商可切换（Ollama / 阿里云百炼 / Gemini）
- **N5.2** TTS 引擎可切换（sherpa-onnx / Piper / 浏览器 SpeechSynthesis）
- **N5.3** STT 引擎可切换（faster-whisper / OpenAI Whisper API）
- **N5.4** Agent 框架可扩展（注册新 Agent 无需修改现有代码）

### F12. 文档上传与解析
- **F12.1** 支持 PDF 和 DOCX 格式的 JD 和简历上传
- **F12.2** 前端提供双区拖拽上传界面（JD + 简历独立区域）
- **F12.3** 后端解析文件文本内容并提取结构化实体
- **F12.4** 文件通过 Base64 编码经 WebSocket 传输

### F13. 实体提取与匹配
- **F13.1** 从 JD 提取岗位名称、技能要求、经验年限、学历要求
- **F13.2** 从简历提取姓名、联系方式、技能、工作经历、教育背景
- **F13.3** 加权技能匹配（必需技能 ×1.5，偏好技能 ×0.5）
- **F13.4** 输出匹配百分比、匹配/缺失技能、技能缺口分析
- **F13.5** 支持 150+ 中文 IT 技能词表 + 60+ 别名映射

### F14. 题库生成
- **F14.1** AI 驱动四类题库生成（破冰、技术、STAR 行为、压力测试）
- **F14.2** 题库包含难度等级（easy/medium/hard）和技能标签
- **F14.3** 题库生成结果通过 WebSocket 实时推送到前端

### F15. 面试进行中
- **F15.1** 基于题库 + JD + 简历 + 匹配结果构建面试专用 instructions
- **F15.2** AI 面试官遵循 STAR 法则进行结构化追问
- **F15.3** 面试对话自动累积为 transcript 供后续评分
- **F15.4** 面试开始/停止消息支持（interview_started / interview_stopped）

### F16. 面试评分与报告
- **F16.1** AI 对面试对话进行 5 维度评分（技术、经验、沟通、匹配、抗压）
- **F16.2** 评分结果以 Canvas 雷达图可视化展示
- **F16.3** 生成结构化面试报告（优势、不足、总结、录用建议）
- **F16.4** AI 解析失败时自动降级到 fallback 报告（基于 match_percentage）
- **F16.5** 报告通过 WebSocket 异步推送到前端（interview_report 消息）

- 对话历史持久化存储
- 自定义 AI 角色/人格
- 拍照快照功能
- 3D Avatar 形象
